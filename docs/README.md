## 테스트 코드 작성 규칙
- 도메인 로직 위주의 단위 테스트를 진행한다.
- 실패 케이스를 먼저 작성한다.
- 직접 구현한 public 메서드는 모두 테스트를 진행한다.
- 메서드는 `@Nested` 어노테이션으로 묶어 각 메서드별 테스트를 분라한다.
- `@DisplayName` 어노테이션으로 한글 설명을 기재한다.
- 메서드명은 위 `@DisplayName`의 간단한 영문이다.
- private method는 테스트를 진행하지 않는다.
- 최종 통합 테스트를 작성한다.
