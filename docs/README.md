## 다리 건너기 미션
```
## BridgeGame
- 입력 받은 문자 "U", "D" 에 따라서 움직일지 판단하는 기능 ⭕️ 
- 게임이 끝났을 때, 재시작 할지 묻는 기능 ⭕️ 
- 올바른 문자가 아닐 경우 재시작 할지 묻는 기능 ⭕️ 
```
```
##BridgeMaker (Controller)
- 다리에 길이를 사용자에게 받고 그만큼 List<String>을 생성 하는 기능 ⭕️ 
- 리스트 안에는 랜덤 숫자에 비교하여, 0 -> "D", 1 -> "U"로 만드는 기능 ⭕️ 
- 랜덤 숫자는 인터페이스 BridgeNumberGenerator에서 상속 받은 기능을 사용하여서 구현 ⭕️ 
```
```
##BridgeDTO
- 게임의 모든 정보를 저장하는 기능 ⭕
- 사용자의 답이 정답이라는 판단이 넘어오면, 다리를 움직이고, 정답 다리를 구현하는 기능 ⭕
- 사용자의 답이 정답이 아니라면, 오답처리를 하는 기능 ⭕
- 처음 게임을 시작할 때와 다시 시도할 때, 사용자의 정답과 오답을 담을 배열 초기화하는 기능 ⭕
- 사용자의 입력이 정답일 경우, 현재 위치를 증가시키는 기능 ⭕
- 모든 정보를 return 해주는 getter ⭕
```
```
##BridgeRandomNumberGenerator
- 랜덤 번호를 설정하여서 만들어주는 기능 ⭕️ 
```
```
##InputView(Model)
- 입력 받은 다리 길이를 int 형으로 변화해주는 기능 ⭕️ 
- 입력 받은 "U", "D"를 리턴해주는 기능 ⭕️ 
- 사용자가 다시 게임을 할지에 대한 "Q", "R"을 입력 받는 기능 ⭕️ 
- 원하는 입력인지 판단하는 validator와 연동하여서 구현 ⭕️
- 모든 입력이 유효하지 않다면, 올바른 값을 입력할 때까지 반복해서 입력을 받는 기능 ⭕
```
```
##OutPutView(View)
- 현재까지 입력받은 올바른 정답으로 move한 다리를 출력해주는 기능 ⭕️ 
- 모든 다리를 이동했다면, 결과를 출력하는 기능 ⭕️ 
- 중간에 오답을 입력했을 경우, 재시작 질문을 출력하는 기능 ⭕️ 
- 이동할 칸을 물어보는 출력 기능 ⭕️ 
- 총 시도한 횟수를 출력하는 기능 ⭕️ 
- 게임을 성공하지 못하고, 종료했을 때 게임 실패 출력하는 기능 ⭕
- 입력 값의 오류일 때 출력해주는 기능 ⭕
```
```
##ValidateUserInput
- 3 ~ 20 사이의 다리길이를 입력했는지 판단하는 기능 ⭕️ 
- 위, 아래를 판단하는 "U", "D"를 입력했는지 판단하는 기능 ⭕️ 
- 재시작, 종료를 판단하는 "R", "Q"를 입력했는지 판단하는 기능 ⭕
```
```
##TotalView
- outputView 와 inputView를 총괄해서 담당하는 기능 ⭕
- Controller에서 정보를 주었을 때, outputview 와 inputview에 전달해주는 기능 ⭕
```
