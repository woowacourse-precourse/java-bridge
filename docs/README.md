# Java - Bridge
이재혁 (JH22222)

1. 기능 구현
   - InputView
     * 다리 길이 세팅 입력
     * 이동할 칸 U / D 선택
     * 게임 재시도 / 종료 선택
   - OutputView
     * 게임 시작시 다리 길이 세팅 요청 출력
     * 게임 단계별 이동 요청 출력
     * 게임 단계별 다리 길이 출력
     * 게임 재도전 여부 요청 출력
     * 게임 클리어 출력
   - BridgeGame
     * 게임 시작
   - BridgeMaker
     * 다리 생성
   - BridgeRandomNumberGenerator
     * 다리 생성시 건널 수 있는 부분 난수로 세팅
2. 프로그래밍 조건
   - indent depth 3 초과 X
   - 3항 연산자 X
   - 함수(또는 매서드)가 한가지 일만 하도록 최대한 최소화
   - 도메인 로직에 단위 테스트 구현
   - 함수(또는 메서드)의 길이가 10라인을 넘지 않도록 한다.
   - 메서드의 파라미터 개수는 최대 3개
   - 주어진 클래스의 요구사항을 참고하여 구연
   - InputView 에서만 입력받을 수 있음
   - BridgeGame 클래스에서 InputView, OutputView 를 사용하지 않는다.
3. 오류 제어 및 테스트케이스
   - 다리 길이 입력시 정수가 아닌 것이 들어올 때
   - 다리길이가 3~20 사이의 값이 아닐 경우
   - 이동할 칸 선택시 U / D 가 아닌 다른 문자가 들어올 경우
   - 재시도 입력시 R / Q 가 아닌 다른 문자가 들어올 경우
